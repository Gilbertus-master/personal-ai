## 4. Przygotowanie na pytania Rocha (Q&A)

### "Kto ma dostep do tych danych?"

> Dzisiaj to jest moja prywatna instancja — tylko ja mam dostep. Docelowo kazda spolka bedzie miala wlasna instancje, ktora nazywam Omnius. Omnius REH bedzie widzial tylko dane REH. Omnius REF tylko dane REF. Zero przeplywu danych miedzy spolkami. Dostep bedzie ograniczony do zarzadu.

### "Czy to jest bezpieczne? Gdzie sa dane?"

> Dane sa na prywatnej infrastrukturze — nie w chmurze. Jedyne co idzie do zewnetrznego API to zapytania do modelu jezykowego (Claude/OpenAI), ale pracuje nad self-hosted LLM, zeby wyeliminowac i te zaleznosc. Dyski sa szyfrowane, dostep przez VPN, audit log na kazdym zapytaniu.

### "Czy pracownicy beda widziec swoje dane?"

> Omnius w wersji firmowej bedzie dzialal na danych organizacyjnych — mailach firmowych, Teamsie, dokumentach na SharePoincie. Przed wdrozeniem potrzebujemy opinii prawnej dot. RODO i polityki monitoringu. Mam to zaplanowane w roadmapie.

### "Ile to kosztuje?"

> Infrastruktura to dedykowany serwer (~500-1000 PLN/msc) plus koszty API (~200-500 PLN/msc na OpenAI/Anthropic). Z self-hosted LLM koszty API spadna do zera. Koszt developmentu to moj czas. Porownaj to z kosztem czasu C-levelu szukajacego informacji w Outlooku.

### "Kiedy to moze dzialac dla REH?"

> MVP Omniusa dla REH moge postawic w 3-4 tygodnie po uzyskaniu: (1) dostepu do Microsoft Graph API dla tenanta REH, (2) serwera w infrastrukturze firmowej, (3) opinii prawnej. Pierwsza wersja bedzie przeszukiwac maile i Teamsa — tak jak widziales dzisiaj.

### "Czy to moze zrobic cos wiecej niz odpowiadac na pytania?"

> Tak — w roadmapie mam: (1) morning brief — codzienne podsumowanie o 7 rano, (2) proaktywne alerty — system sam informuje o zalegych follow-upach, (3) command protocol — Gilbertus moze zlecac zadania Omniusowi (np. wyslij maila, stworz ticket), ale tylko po zatwierdzeniu przez czlowieka. Zawsze human-in-the-loop.

### "A co z Krystianem / REF?"

> Omnius REF to osobna instancja — identyczna architektura, ale kompletnie oddzielone dane. Omnius REH nie widzi danych REF i odwrotnie. Moge postawic obie instancje rownolegle.

### "Czym to sie rozni od Copilota / ChatGPT?"

> Trzy kluczowe roznice: (1) Dane — Gilbertus dziala na NASZYCH danych firmowych, nie na internecie. (2) Kontekst — system rozumie encje, relacje, timeline — nie tylko wyszukuje slowa kluczowe. (3) Prywatnosc — dane nie opuszczaja naszej infrastruktury, w przeciwienstwie do Copilota ktory przetwarza dane w chmurze Microsoftu.

## 5. Setup techniczny — checklist przed prezentacja

### 30 minut przed demo

- [ ] Uruchom infrastrukture: `./scripts/run_infra.sh` (Postgres + Qdrant)
- [ ] Uruchom API: `./scripts/run_api.sh`
- [ ] Sprawdz zdrowie: `curl http://127.0.0.1:8000/health`
- [ ] Sprawdz czy presentation endpoint dziala: `curl http://127.0.0.1:8000/presentation/intro`
- [ ] Otworz ngrok/tunel jesli demo jest zdalne: `ngrok http 8000`
- [ ] Otworz przegladarke na stronie demo: `http://127.0.0.1:8000/presentation/demo`

### Przetestuj pytania przed prezentacja

Uruchom kazde z 10 pytan i upewnij sie, ze odpowiedzi sa sensowne. Jesli ktores nie dziala:
- Sprawdz logi: `tail -f logs/api.log`
- Sprawdz czy Qdrant ma dane: `curl http://localhost:6333/collections`
- Sprawdz czy source_types sa poprawne w filtrze prezentacyjnym

### Co miec otwarte na ekranie

1. **Glowne okno:** strona demo (`/presentation/demo`) — pelny ekran
2. **Zapasowe okno:** terminal z logami API (na wypadek bledu)
3. **Notatnik:** ten dokument z lista pytan do skopiowania

### Na wypadek awarii

- Jesli API nie odpowiada: `./scripts/run_api.sh` restart
- Jesli odpowiedz jest pusta: poszerzenie `top_k` w kodzie lub zmiana pytania
- Jesli odpowiedz jest za wolna (>15s): wspomnij ze to MVP na laptopie, docelowo dedykowany serwer
- **Plan B:** Jesli nic nie dziala — miej screenshoty przygotowanych odpowiedzi w osobnym folderze

### Ustawienia prezentacji

- **TTS wylaczony** domyslnie — wlacz tylko jesli Roch chce uslyszec odpowiedz glosowa (fajny efekt wow, ale spowalnia)
- **Rozdzielczosc ekranu:** ciemny motyw wyglada dobrze na projektorze, ale zwieksz font w przegladarce (Ctrl+)
- **Mikrofon:** przetestuj rozpoznawanie mowy po polsku jesli chcesz pokazac voice input
